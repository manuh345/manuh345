#### [Ties — Relationship Verification Platform](https://github.com/emmanuelbett/ties)
> Node.js · Express · MongoDB · JWT · HTML/CSS

A peer-based verification system where 5 people must vouch for a relationship
before a signed certificate gets issued. Built the whole thing: auth, workflow
logic, certificate generation, responsive UI.

What I figured out along the way:
- Designing multi-step approval state machines
- JWT auth + session management across front and back
- MongoDB schema design for relationships and approval records

#### [Cyber Café Management System](https://github.com/emmanuelbett/cafe-manager)
> Node.js · MySQL · JavaScript

Session tracking, billing, and admin dashboard for a cyber café. Built for
non-technical operators — the UI had to just work without explanation.
